Statistical analysis, data cleaning pipeline, and outlier treatment for the EverPeak-SilverBasket retail case study.

# EverPeak Retail Analysis – Sprint 6

This repository contains the analysis performed during the Sprint 7 of the EverPeak–SilverBasket case study.

The dataset `everpeak_retail` includes 2,000 customer orders containing missing values, sentinels, outliers, and data quality issues designed to simulate real-world retail data.

## 📂 Repository Content

- `everpeak_analysis.ipynb`  
  → Main Jupyter notebook containing data cleaning pipelines, EDA, distributions, outlier treatment, and final insights.

## 📘 How to Reproduce the Analysis

1. Open the `everpeak_analysis.ipynb` file.
2. Run the code cells sequentially from top to bottom.
3. The notebook handles dataset loading automatically.

## 🧠 Analysis Objectives

- Identify and flag data quality and structural issues.
- Build a robust, reproducible data cleaning pipeline.
- Analyze consumer behavior trends, statistical distributions, and extreme outliers.
- Provide clear business insights for the EverPeak Strategy and Integration team.
